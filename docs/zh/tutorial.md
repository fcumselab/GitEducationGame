# 🎮 GITainment 專案介紹及部署教學

歡迎加入本專案！這是一個結合了 **Unity 遊戲前端**、**Node.js 邏輯後端** 與 **MongoDB 資料庫** 的全端應用。本指南將帶領你從零開始配置環境，直到最後成功部署。

---

## 🗺️ 專案藍圖 (Architecture)

在動手之前，我們先用一張簡單的藍圖來理解這三個組件是如何互相溝通的：

- **🎮 前端 (Unity):** 負責呈現遊戲畫面與玩家的操作。當玩家進行特定行為（例如：玩家登入、儲存通關資料、查詢排行榜）時，會向後端發送網路請求。
- **🧠 後端 (Node.js):** 作為遊戲的伺服器，負責處理遊戲邏輯、驗證玩家資料，並與資料庫進行溝通。
- **💾 資料庫 (MongoDB):** 負責儲存玩家的各項核心資料（如：帳號密碼、遊戲進度）以及全球遊戲排行榜的資料。

---

## 🛠️ 第一階段：獲取專案與環境結構介紹

首先，請先將專案複製到你的本機環境：

```
git clone https://github.com/kevin38645/GitEducationGame.git
```

成功複製後，你會看到專案內包含以下四個主要核心資料夾：

| **資料夾名稱** | **用途**       | **說明與注意事項**                                                |
| -------------- | -------------- | ----------------------------------------------------------------- |
| **`docker/`**  | 環境架設       | 包含 `docker-compose.yml`，用於後期一鍵架設與部署整個遊戲生態系。 |
| **`server/`**  | 後端服務       | Node.js 專案，作為遊戲服務器使用，處理所有 API 請求。             |
| **`game/`**    | Unity 原始碼   | Unity 遊戲專案資料夾，用於開發、修改遊戲內容與視覺。              |
| **`gegGame/`** | WebGL 靜態網頁 | Unity 實際打包（Build）出來的 WebGL 遊戲文件。                    |

> ⚠️ **提醒：**
> `gegGame/` 資料夾內的文件**可能不是最新版本的**。
> 如果你修改了 `game/` 資料夾內的任何遊戲內容，請務必記得使用 Unity 重新 Build 出 WebGL 檔案
> ，並覆蓋（更新）到這個資料夾中，這樣網頁端才會呈現最新的遊戲畫面

為了讓你順利跑起整個專案，請按照順序閱讀並配置以下三份環境教學文件：

1.  **[💾 MongoDB 資料庫配置指南](tutorial-mongodb.md)**：學習如何在本機啟動資料庫、設定連接字串。
2.  **[🧠 Node.js 後端環境架設指南](tutorial-nodejs.md)**：學習如何安裝 Node.js、套件，並讓伺服器在本地跑起來。
3.  **[🎮 Unity 前端開發環境配置](tutorial-unity.md)**：引導你下載對應的 Unity 版本，並成功開啟遊戲專案與修復異常。

當你閱讀並成功建立運行良好的專案環境時，你應該已經可以在本地電腦上順利進行遊戲開發與測試了。
