不過，在 "提交記錄" 視窗中
請注意 "修改顏色" 分支與 "master" 分支的 "提交記錄"
它們分成了兩個不同的路徑

在這種情況下合併分支時
Git 會採用自動合併模式

Git 會比對兩個分支修改的內容
並發現在修改文章顏色的文件中出現了內容衝突

由於 Git 無法去自行判斷文字的顏色
應該是 "master" 分支的藍色
還是 "修改顏色" 分支的橙色

在執行了 "git merge 分支名稱" 指令後
Git 會提醒您在合併時出現了文件衝突

那麼，現在讓我們來使用 "git merge 修改顏色" 指令
嘗試合併分支吧

在執行過後，請先確認 "提交記錄" 的狀態
